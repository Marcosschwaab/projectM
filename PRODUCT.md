# Product

## Register

product

## Users

Project managers/team leads and team members alike. PMs need progress overview, reports, OKR/KPI tracking, and Gantt/calendar planning. Contributors need task management, kanban, comments, and checklists. Both groups use the tool daily inside a browser, in a work context — often with multiple projects and cross-functional teams.

## Product Purpose

projectM is a modern project management tool that replaces heavy enterprise tools (Jira, Salesforce) with a clean, focused experience. It helps teams plan, track, and deliver work through OKRs, KPIs, tasks, timelines, and financial oversight — without the clutter. Success means teams spend less time managing the tool and more time doing the work.

## Brand Personality

Clean, professional, calm. The tool should feel organized and trustworthy — it gets out of the way and lets the work speak. Not flashy, not cold. Purposeful in every element.

## Anti-references

- Jira / Salesforce / heavy enterprise tools: cluttered interfaces, dense data without hierarchy, overwhelming number of controls per screen.
- Generic Bootstrap admin templates: no visual identity, everything looks the same.

## Design Principles

1. **The tool disappears.** Every screen should let the user focus on their work, not the interface. Purposeful reduction.
2. **Familiarity through consistency.** Same component vocabulary everywhere — one button, one form control, one navigation pattern. Users should never guess.
3. **Calm density.** Information-rich without feeling busy. Good hierarchy, generous whitespace where it matters, compact data where it serves.
4. **State-rich, not decoration-rich.** Color, motion, and visual weight convey meaning (status, priority, progress) — never decoration.

## Accessibility & Inclusion

Target WCAG 2.1 AA. Minimum 4.5:1 contrast for body text, 3:1 for large text. Keyboard navigable. Proper focus states. Reduced motion support for all transitions.
